# 학습 효과 강화 기능 분석

자녀용 수학 게임에 "**가르치기 (teaching)**" 레이어를 추가할 후보 분석.
결정 후 채택된 항목은 `ROADMAP.md`로 이관.

작성: 2026-06-08 / 기준 버전: 1.3.0+9

---

## 1. 현재 앱의 학습 메커니즘 진단

### 잘 갖춰진 영역 — **동기부여 · 평가 · 측정**

| 축 | 현재 구현 |
|---|---|
| 재미 / 동기 | 도장 54+개, 콤보 시스템, 데일리 미션 3개/일, streak 카운트, 추천 카드 |
| 평가 / 측정 | 약점 분석 (`weakness.dart`), 통계 화면, 결과 기록, 오답 노트(dismiss 포함) |
| 문제 다양성 | 4연산 × 5레벨, 구구단, 혼합, 방정식, 플래시, 도전/연습/타임어택 |
| 인터랙션 | SFX/햅틱, Lottie, 결과 상세, 커스텀 도장 |

### 비어 있는 영역 — **티칭(teaching)**

거의 모든 시스템이 "평가 → 보상" 구조에 집중되어 있음. 다음 두 가지가 명백히 빠져 있음:

1. **"왜 틀렸는지 / 어떻게 풀어야 했는지"를 짚어주는 설명 레이어**
   현재 오답에 대한 피드백은 `correct.wav`/`wrong.wav` SFX + 결과 화면의 정답 표시뿐.
2. **틀린 문제를 "잊지 않도록" 자동으로 다시 만나게 하는 메커니즘**
   오답 노트가 있지만 사용자가 능동적으로 "다시 풀기"를 눌러야 함. 망각 곡선과 맞물리지 않음.

이 두 빈 칸을 채우는 게 학습 효과 면에서 가장 큰 도약.

## 2. 후보 5가지

각 후보는 임팩트(학습 효과) × 공수(구현 비용) × 적합성(이 앱·이 연령대)으로 평가.

### 후보 A. 풀이 과정 표시 (Step-by-step Solution)

**무엇**: 오답 시 정답만 보여주는 대신, 어떻게 풀어야 했는지 단계로 풀어서 표시.
- 덧셈 `23 + 19`: "일의 자리 3+9=12 (받아올림 1) → 십의 자리 2+1+1=4 → **42**"
- 뺄셈 `52 - 27`: "일의 자리 2-7 불가 → 십의 자리에서 1 빌림 → 12-7=5, 4-2=2 → **25**"
- 곱셈 `12 × 3`: "10×3=30, 2×3=6, 합 → **36**"
- 나눗셈 `36 ÷ 4`: "4×9=36이므로 → **9**"

**임팩트**: ⭐⭐⭐⭐⭐ — 받아올림·받아내림은 초등 수학에서 가장 큰 걸림돌. 정답만 보면 다음에 또 틀림.

**공수**: 중간 — 4연산 각각의 풀이 생성 알고리즘이 필요하지만 순수 함수. 외부 콘텐츠/디자인 자산 불필요.
- `lib/app/shared/solution.dart` (신규) — 입력 `ProblemAttempt` → 출력 `List<SolutionStep>`.
- `attempt_tile.dart` 또는 `record_detail`/`review` 화면에 펼침/접힘 토글.

**적합성**: 매우 높음 — 기존 오답 노트와 자연 결합 (오답 항목 탭 → 풀이 보기 → 다시 풀기 옵션).

**리스크**: 혼합/방정식 모드에서 풀이가 복잡해질 수 있음 → 1차 릴리스는 단일 op만 지원, 혼합은 v2.

---

### 후보 B. 간격 반복 (Spaced Repetition)

**무엇**: 오답 노트 항목을 다음 게임에 일정 비율 자동 혼입. 망각 곡선 기반 간격(1d/3d/7d/14d)으로 재등장.

**임팩트**: ⭐⭐⭐⭐ — Anki·Duolingo가 쓰는 검증된 메커니즘. 장기 기억 정착.

**공수**: 중간 —
- 시그니처별 "다음 등장 시각" 필드 추가 (이미 만든 `wrongNotebookSignature` 재활용).
- `ProblemGenerator`에 "큐에서 X% 확률로 뽑기" 옵션 추가.
- 정답 시 다음 간격 증가, 오답 시 간격 리셋.

**적합성**: 매우 높음 — 기존 오답 노트 + `RecordService` 위에 얹는 구조.

**ROADMAP 기존 항목 8 ("누적 오답 드릴 모드")과의 관계**: 항목 8은 "오답만 풀기" 별도 모드(능동), 본 후보는 "일반 게임에 오답 자동 섞기"(수동). **상호 보완 — 같이 하면 학습 사이클 양 방향 모두 커버**.

**리스크**: 너무 자주 섞이면 "이거 또야?" 피로감. 비율 조절(예: 10문제 중 1~2개) + 사용자 토글 옵션.

---

### 후보 C. 적응형 난이도 (Adaptive Difficulty)

**무엇**: 세션 내에서 연속 정답이면 자릿수가 슬쩍 올라가고, 연속 오답이면 잠시 내려감. 항상 flow zone에 머무름.

**임팩트**: ⭐⭐⭐ — 지루함·좌절 양쪽을 줄임. 다만 이미 5단계 + 약점 추천이 있어 가산 효과는 A·B보다 작음.

**공수**: 중간 — `ProblemGenerator`에 동적 입력 추가. 기존 5단계와 별도로 "혼합 난이도" 모드를 신설하는 게 깔끔.

**적합성**: 중간 — 새 모드 1개 추가. 기존 시스템과 직교.

**리스크**: 알고리즘이 너무 보수적이면 변화를 못 느끼고, 너무 공격적이면 흔들림. 튜닝 비용.

---

### 후보 D. 시각적 표현 (Visual Number Sense)

**무엇**: 추상 숫자를 직관 가능한 그림으로 보여줌.
- 덧셈/뺄셈: 수직선 위 점프
- 곱셈: 격자(array) 또는 묶음
- 나눗셈: 사과 N개를 M개씩 묶기

**임팩트**: ⭐⭐⭐⭐ (저학년) — 1·2학년에 특히 강력. 추상 → 직관 다리.

**공수**: 중-높음 —
- 4연산 각각 CustomPainter 또는 SVG/Lottie.
- 디자인 결정 많음 (어디 표시, 언제 표시, 토글 여부).

**적합성**: 자녀 연령에 의존. 저학년이면 최우선, 고학년이면 보조.

**리스크**: 디자인 작업 비중이 코드보다 큼.

---

### 후보 E. 문장제 (Word Problems)

**무엇**: "사과가 3개 있는데 5개 더 사면?" 형식. 학교 시험·실생활 핵심 영역.

**임팩트**: ⭐⭐⭐⭐⭐ (교과 일치도) — 진짜 교과서 영역.

**공수**: 매우 높음 —
- 한국어 문장 템플릿 풀.
- 명사 사전 (개/명/마리/원), 조사 규칙 (이/가, 을/를).
- 상황·맥락 데이터.

**적합성**: 매우 높지만 콘텐츠 작업이 코드보다 큰 비중. 별도 phase 권장.

**ROADMAP 기존 항목 6 ("문장제 문제")**과 동일 — 이미 대기 중.

## 3. 비교 요약

| 후보 | 학습 임팩트 | 공수 | 기존 자산 재활용 | 즉시 체감도 | 권장 우선순위 |
|---|---|---|---|---|---|
| A. 풀이 과정 | ⭐⭐⭐⭐⭐ | 중 | 오답 노트 결합 | 매우 높음 | **1순위** |
| B. 간격 반복 | ⭐⭐⭐⭐ | 중 | 시그니처/노트/서비스 | 중-높음 (누적) | **2순위** |
| C. 적응형 난이도 | ⭐⭐⭐ | 중 | 적음 (새 모드) | 낮음 (점진적) | 3순위 |
| D. 시각화 | ⭐⭐⭐⭐ (저학년) | 중-높 | 적음 | 높음 (즉시) | 연령 의존 |
| E. 문장제 | ⭐⭐⭐⭐⭐ | 매우 높 | 적음 | 높음 | 별도 phase |

## 4. 권장 진행 순서

### 단기 (다음 릴리스)
**A. 풀이 과정 + B. 간격 반복을 묶음 배포.**

근거:
- **학습 사이클의 두 축이 채워짐**: A는 "가르치기", B는 "잊지 않기" — 한 쌍.
- **기존 자산 100% 재활용**: 오답 노트 시그니처·`RecordService`·문제 생성기 모두 그대로 위에 얹음.
- **공수 합리적**: 둘 다 순수 함수+서비스 확장 위주. 새 화면 1-2개 추가 수준.
- **즉시 체감 + 누적 효과 동시**: A는 첫 오답에서 바로 효과, B는 일주일 후부터 누적.

### 중기
**D. 시각화** — 자녀 연령 확인 후 결정. 저학년이면 우선 처리, 고학년이면 보조 토글.
**C. 적응형 난이도** — 별도 모드로 가볍게 추가.

### 장기
**E. 문장제 (ROADMAP 6번)** — 콘텐츠 작업이 큰 비중이므로 별도 phase.

## 5. 결정해야 할 것

권장안(A + B)으로 가는 경우 1차 설계 단계에서 잡아야 할 결정:

### 풀이 과정 (A)
- [ ] 표시 위치: 오답 즉시 (게임 중) vs 결과 화면 vs 오답 노트 상세
- [ ] 표시 형태: 텍스트 단계 vs 시각적 표현 결합 (D와 묶을지)
- [ ] 혼합/방정식 모드 지원 여부 (v1에서는 단일 op만?)
- [ ] "풀이 보기" 버튼을 명시적으로 누르게 할지 / 자동 펼침인지

### 간격 반복 (B)
- [ ] 혼입 비율 (10문제 중 1·2·3개 중 어느 정도?)
- [ ] 간격 (1d/3d/7d/14d 표준 vs 더 짧게)
- [ ] 정답 시 그래도 다시 등장시킬지 (확실히 익혔다는 기준)
- [ ] 사용자(부모) 토글 옵션 제공 여부
- [ ] 도전 모드만 vs 연습 모드 포함

설계 단계로 진입할 때 위 결정사항부터 합의.
