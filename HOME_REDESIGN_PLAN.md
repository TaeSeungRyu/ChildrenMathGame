# 홈 화면 개편안 (게임모드 추가 대비)

> 게임모드(몬스터 처치 / 풍선 터뜨리기 / 타워 디펜스 / 두더지 잡기) 4개를 새로 들이면서, 기존 콘텐츠를 유지·재배치한다.
> 목적: **6~9세 아동이 한눈에 어디로 갈지 알 수 있는 구조**를 유지하면서 콘텐츠 슬롯을 늘린다.

---

## 현재 홈에 담긴 것 (참고)

- AppBar: 이름 편집 가능한 타이틀, 튜토리얼 / 음소거
- 본문 (위→아래)
  1. Lottie 배너 + 스트릭 뱃지
  2. 오늘의 미션 카드
  3. (조건부) 약점 추천 카드
  4. "기본 연산" 섹션 - 2×2 그리드 (덧셈/뺄셈/곱셈/나눗셈)
  5. "특별 모드" 섹션 - 1×4 행 (구구단/혼합/방정식/플래시)
- BottomBar: 도장 / 오답 / 기록

→ 여기에 **게임 4개**를 더 얹어야 하고, 향후 통계/복습/뱃지 등도 접근성을 잃지 않아야 한다.

---

## 개편안 후보

### 안 1. 바텀 탭 네비게이션 ⭐ 1차 추천

```
┌─────────────────────────┐
│  ○○○ 히어로!     ❓ 🔊  │  AppBar
├─────────────────────────┤
│  [현재 탭 본문]          │
│                         │
│                         │
├─────────────────────────┤
│  📚학습  ⚔️게임  📊기록  │  BottomNav (3탭)
└─────────────────────────┘
```

- **학습 탭**: 지금의 홈을 그대로 유지 (미션 / 추천 / 기본 연산 4 / 특별 모드 4).
- **게임 탭**: 신규. 게임모드 4개 타일 + 게임 전용 누적 기록(최고 처치 수, 최고 콤보 등).
- **기록 탭**: 지금의 하단 도장/오답/기록을 통합한 메타 허브. 도장·오답·기록·통계·복습을 한 화면 안의 카드들로.

**장점**
- 가장 명확한 구조 분리. "학습"과 "게임"이 시각·동선상 완전히 분리됨.
- 각 탭이 단순해져 추가 콘텐츠를 더 들여도 부담이 적음.
- BottomBar라는 패턴은 Material 표준이라 아동도 학습됨.

**단점**
- 현재 홈에서 곧장 보이던 "오늘의 미션 / 추천"이 학습 탭으로 들어가지만, 진입 즉시 학습 탭이 기본이라 실질 변화는 작다.
- BottomNav 도입은 라우팅 구조 변경(홈이 단일 Scaffold → 탭 컨테이너)이 필요.

### 안 2. 카테고리 허브 (드릴다운)

```
┌─────────────────────────┐
│  ○○○ 히어로!     ❓ 🔊  │
├─────────────────────────┤
│  [오늘의 미션 카드]      │
│  [추천 카드 - 있을 때]   │
│                         │
│  ┌──────────┬──────────┐│
│  │ 📚 학습  │ ⚔️ 게임  ││  큰 타일 2개
│  └──────────┴──────────┘│
│                         │
│  도장  오답  기록  통계  │  하단 메타 4
└─────────────────────────┘
```

- 홈은 "학습 / 게임" 2개 큰 카테고리 카드만 노출.
- "학습" 탭 → 기본 연산 4 + 특별 모드 4 (현재 본문)
- "게임" 탭 → 게임모드 4
- 하단 메타: 도장 / 오답 / 기록 / 통계 (4개로 확장)

**장점**
- 홈이 가장 깔끔. 6~7세가 보기에 결정 부담이 최소.
- 메인 카테고리 추가가 쉬움(나중에 "친구 대전" 같은 게 와도 카드 하나 추가).

**단점**
- 가장 자주 쓰는 "기본 연산"이 한 탭 더 들어가서 도달 깊이 +1.
- 학습 탭과 게임 탭 진입이 모두 새 화면 push라 BottomNav보다 동선이 길다.

### 안 3. 단일 스크롤 + 섹션 추가 (최소 개편)

```
┌─────────────────────────┐
│  [배너 + 스트릭]         │
│  [미션 카드]             │
│  [추천 카드]             │
│  ──────────────         │
│  🧮 기본 연산            │
│  [+][-][×][÷]           │
│  ──────────────         │
│  ⭐ 특별 모드            │
│  [구구단][혼합][방정식][플래시]│
│  ──────────────         │
│  ⚔️ 게임모드   ← 신규    │
│  [몬스터][풍선][디펜스][두더지]│
└─────────────────────────┘
│  도장  오답  기록        │
└─────────────────────────┘
```

- 현재 구조 유지, "게임모드" 섹션만 아래에 추가.
- BottomBar는 그대로.

**장점**
- 구현 부담 최소. 라우팅·바인딩 변경 없음.
- 모든 모드가 한 스크롤 안에 있어 탐색이 직관적.

**단점**
- 화면이 길어져 스크롤이 필수가 됨(특히 작은 안드로이드 폰).
- 콘텐츠가 더 추가되면 다시 개편이 필요해짐(미래 부채).
- 시각적 밀도가 너무 높아 6~7세에게 부담.

### 안 4. 가로 스와이프 페이지 (PageView)

```
┌─────────────────────────┐
│  ○○○ 히어로!     ❓ 🔊  │
├─────────────────────────┤
│  ● ○ ○        (인디케이터)│
│                         │
│   [현재 페이지 본문]     │
│   ← 학습 | 게임 | 기록 → │
│                         │
└─────────────────────────┘
```

- 좌→우 스와이프로 3페이지(학습 / 게임 / 기록) 전환.
- 인디케이터 점 3개.

**장점**
- BottomNav 없이도 분리 효과.
- 스와이프 인터랙션이 아이들에게 재미 요소.

**단점**
- 페이지 존재를 모르는 사용자가 발견하기 어려움(어포던스 약함).
- 인디케이터만으로는 6~7세가 다른 페이지 의미를 알기 힘듦.
- 실수로 스와이프해 길을 잃을 가능성.

## 1차 추천: 안 1 (바텀 탭 네비게이션)

### 추천 이유

1. **"학습 vs 게임"의 분리**가 가장 중요한 컨셉 변화이고, 이걸 가장 강하게 보여주는 게 BottomNav.
2. 현재 홈은 이미 빽빽해서 "특별 모드" 아래에 게임모드 4개를 더 얹으면 시각적 한계를 넘는다(안 3 탈락 이유).
3. BottomNav는 6~9세 아동에게도 매우 익숙한 UI 패턴(YouTube Kids 등).
4. 게임 탭은 "Time Attack 최고 콤보" 같은 게임 전용 누적 지표를 첫 화면에 띄울 수 있어, 동기부여 장치를 따로 분리해 운영할 수 있다.

### 탭 구성 (제안)

| 탭 | 아이콘 | 본문 |
|----|-------|------|
| 학습 | 📚 (book) | 미션 / 추천 / 기본 연산 4 / 특별 모드 4 (= 현재 홈) |
| 게임 | ⚔️ (sword) | 게임 4 타일 + "최고 처치", "최고 콤보" 같은 누적 지표 카드 |
| 기록 | 📊 (chart) | 도장·오답·기록·통계·복습 (5개 카드) |

### 구현 시 영향 범위

```
lib/app/modules/home/
  home_view.dart            (재작성: Scaffold + BottomNavBar + 내부 IndexedStack)
  home_controller.dart      (수정: currentTab Rx 추가, 기존 콜백 유지)
  home_binding.dart         (그대로)
  tabs/                     (신규)
    learn_tab.dart          (= 현재 홈 본문)
    games_tab.dart          (신규)
    records_tab.dart        (현재 BottomBar에 있던 quick action들을 카드로 승격)
```

- 기존 라우트(`AppRoutes.records`, `badges`, `wrongNotebook`, `stats`, `review`)는 그대로 유지하고, 기록 탭에서 카드 탭 → 해당 라우트로 이동.
- 게임모드는 이후 단계에서 추가될 때 `lib/app/modules/action_*` 모듈로 들어오고, 게임 탭 안의 타일이 해당 라우트로 연결.

### 게임 탭의 점진적 채움

- 1차: 게임 4개 자리만 만들고, 미구현 타일은 "곧 만나요" 상태로 표시(잠금 자물쇠 아이콘 + 살짝 흐리게).
- 2차~5차: 컨셉 하나씩 구현되면 타일이 활성화. 진행 상황이 한눈에 보여 동기부여도 됨.

```
⚔️ 게임모드

┌──────────┬──────────┐
│ 🦸 몬스터 │ 🎈 풍선   │   ← 활성
│  처치    │  터뜨리기 │
├──────────┼──────────┤
│ 🏰 타워   │ 🕳️ 두더지 │   ← 잠금 (아직 미구현)
│  디펜스  │  잡기    │
└──────────┴──────────┘
```

## 결정 필요 항목

1. **개편안 확정**: 1 / 2 / 3 / 4 중 하나.
2. **탭 구성**: 추천한 3탭(학습·게임·기록)이 적절한지, 아니면 2탭(학습+게임, 기록 메뉴는 AppBar로)이 나은지.
3. **게임 탭 미구현 처리**: "잠금 + 흐리게" / "Coming soon 배지" / "타일 자체를 숨김" 중 선호.
4. **게임 우선순위**: 어느 컨셉부터 만들지 (A→B→C→D 순? 학습 효과 강한 B를 먼저? 등).
