# 게임모드(액션) 기획안

> 프로젝트명이 **연산 히어로**로 바뀐 만큼, "히어로가 연산으로 무언가를 처치/돌파한다"는 액션성을 담은 새 모드를 추가한다.
> 기존 기능(기본 연산 / 구구단 / 혼합 / 방정식 / 플래시 / 도장 / 오답 / 기록)은 그대로 유지하고, **홈의 "특별 모드" 줄에 항목 한 칸을 추가**하는 형태.

---

## 공통 전제

- **위치**: 홈 화면 → "특별 모드" 줄에 새 타일 `⚔️ 게임` 또는 `🦸 히어로` 추가.
- **세션 형태**: 기존 challenge/practice/timeAttack과 **별도 모드**. `SessionMode.action` 추가.
- **문제 출제**: 기존 `ProblemGenerator`를 그대로 재사용 (아래 "공통 진입 선택"에서 고른 연산 + 자릿수 조합으로 생성).
- **종료 조건**: 액션 컨셉마다 다름(아래 참조). 종료 시 `GameRecord`에 `mode = action`으로 저장.
- **퍼펙트/마스터 뱃지**: 기존 challenge만 측정한다는 규약은 유지. 액션 모드는 누적 카운트(정답 합계 / 콤보 / 스트릭)에만 기여.
- **진입 선택 화면**: 모든 액션 컨셉(A/B/C/D)에서 동일한 "공통 진입 선택" 화면을 공유한다(아래 참조). 기존 `LevelSelectView`는 challenge 전용이므로 재사용하지 않고, 액션 전용 셀렉트 화면을 신설한다.

---

## 공통 진입 선택 (모든 액션 컨셉 공유)

홈 → `⚔️ 게임` 타일을 누르면 항상 같은 진입 화면이 뜬다. 여기서 **연산 종류**와 **자릿수 조합**을 고른 뒤 액션 컨셉(A~D 중 선택된 것) 본편으로 입장한다.

### 1) 연산 종류 (5지선다, 단일 선택)

| 옵션 | 동작 |
|------|------|
| ➕ 덧셈 | `GameType.addition` 고정으로 출제 |
| ➖ 뺄셈 | `GameType.subtraction` 고정으로 출제 (결과 ≥ 0 보장) |
| ✖️ 곱셈 | `GameType.multiplication` 고정으로 출제 |
| ➗ 나눗셈 | `GameType.division` 고정으로 출제 (나누어떨어지는 식) |
| 🎲 랜덤 | 문제 출제 시마다 4개 중 무작위로 선택 — 한 세션 안에서 연산이 섞임 |

### 2) 자릿수 조합 (5지선다, 단일 선택)

기존 `_digitsForLevel` 매핑을 **사용자가 직접 고르는 형태**로 노출한다 (레벨 번호 대신 자릿수 표기로 표시).

| 라벨 | (A자릿수, B자릿수) | 비고 |
|------|------------------|------|
| 1자리 × 1자리 | (1, 1) | 가장 쉬움 |
| 2자리 × 1자리 | (2, 1) | |
| 2자리 × 2자리 | (2, 2) | |
| 3자리 × 2자리 | (3, 2) | |
| 3자리 × 3자리 | (3, 3) | 가장 어려움 |

- 연산이 **나눗셈**인 경우: A = 나눠지는 수 자릿수, B = 나누는 수 자릿수. 기존 `ProblemGenerator` 규칙(1자리 제수는 2~9, 몫 ≥ 2) 그대로 유지.
- 연산이 **랜덤**인 경우: 자릿수 조합은 그대로 적용하되, 매 문제마다 연산만 무작위로 뽑는다.

### 3) 인자 전달

선택 결과는 `Get.toNamed(Routes.actionGame, arguments: {...})`로 다음과 같이 전달:

```dart
{
  'gameType': GameType.addition,   // 또는 null(=랜덤)
  'digitsA': 2,                    // 1, 2, 3 중 하나
  'digitsB': 1,                    // 1, 2, 3 중 하나
  'concept': ActionConcept.monster // A/B/C/D 중 어느 컨셉인지
}
```

- `gameType == null`이면 `ProblemGenerator`가 매 문제 `GameType.values`에서 무작위 추출.
- `digitsA`/`digitsB`는 기존 `_digitsForLevel`을 우회해 직접 주입하는 새 진입점이 필요 → `ProblemGenerator.generateForDigits(type, digitsA, digitsB)` (신규 메서드) 또는 `generate()`에 옵셔널 자릿수 파라미터를 추가한다.

### 4) UI 스케치

```
─────────────────────────────
        ⚔️ 게임 시작

   연산을 골라요
   [ ➕ ] [ ➖ ] [ ✖️ ] [ ➗ ] [ 🎲 ]
            ↑선택됨

   자릿수를 골라요
   [ 1×1 ] [ 2×1 ] [ 2×2 ] [ 3×2 ] [ 3×3 ]
                       ↑선택됨

        [   시작하기   ]
─────────────────────────────
```

- 두 항목 모두 선택돼야 "시작하기" 버튼 활성화.
- 마지막 선택값은 `SharedPreferences`에 저장해 다음 진입 때 기본 선택으로 복원 (키: `action_last_op_v1`, `action_last_digits_v1`).

## 컨셉 후보 4가지

### A. 몬스터 처치 (히어로 vs 몬스터) ⭐ 메인 추천

```
   🦸 HP ❤️❤️❤️
─────────────────
        👹
      [ 7 × 8 ]
        ↓
─────────────────
    [입력칸]
   [숫자패드]
```

- **흐름**: 화면 위에서 몬스터가 문제를 들고 천천히 내려옴 → 정답 입력 → 공격 애니메이션 + 몬스터 처치 → 다음 몬스터 등장.
- **실패 처리**: 몬스터가 바닥에 닿거나 오답 입력 → 히어로 HP -1. HP 3개 소진 시 게임오버.
- **난이도 상승**: 처치 수가 늘수록 몬스터 낙하 속도 ↑, 동시 등장 수 ↑(웨이브 형식 가능).
- **보상감**: 콤보 카운트, 처치 수, 보스 몬스터(특정 수치) 등장.
- **장점**
  - 프로젝트명 "히어로"와 가장 직관적으로 연결됨.
  - 시간 압박이 자연스럽게 게임에 녹아들어, 별도의 카운트다운 UI가 필요 없음.
  - Lottie 애니메이션(공격/처치 이펙트)이 이미 있는 자산 흐름과 잘 맞음.
- **단점/리스크**
  - 몬스터 낙하 애니메이션 + 처치 이펙트 등 **신규 그래픽 자산이 가장 많이 필요**.
  - 아동용 톤 유지 위해 몬스터 디자인은 귀여운 쪽으로 고정 필요(공포감 X).

---

### B. 풍선 터뜨리기 (밸런스 게임)

```
  목표: 답 = 42

  🎈37   🎈42   🎈19
  6×6    7×6    3×7

  🎈21   🎈42
  3×7    6×7
```

- **흐름**: 화면에 여러 문제 풍선이 떠 있고, 상단에 "목표 답" 제시 → 답이 일치하는 풍선만 골라서 탭 → 정답 풍선 모두 터뜨리면 다음 라운드.
- **실패 처리**: 오답 풍선을 터뜨리면 페널티(시간 감소 또는 점수 차감). 시간 제한 종료 시 게임오버.
- **난이도 상승**: 라운드당 풍선 수 ↑, 정답 풍선 개수 ↑, 풍선 이동 속도 ↑.
- **장점**
  - 그래픽 자산이 단순(풍선 + 터지는 이펙트).
  - "암산 → 빠른 선택"이라는 두뇌훈련 색이 강함.
- **단점**
  - "히어로" 테마와의 연결이 약함(풍선 → 히어로 정체성?).
  - 한 화면에 수식이 너무 많으면 6~8세에게 시각적 부담.

---

### C. 타워 디펜스 (마법 공격)

```
🏰━━━━━━━━ 👹👹👹
      ↑
   ⚡ 공격!
─────────────
  [ 12 + 8 ]
   [입력]
```

- **흐름**: 화면 오른쪽에서 몬스터들이 성(왼쪽)을 향해 행진 → 문제를 풀면 마법/화살 공격 발사 → 몬스터 처치.
- **실패 처리**: 몬스터가 성에 닿으면 성 HP -1. 성 HP 3 소진 시 게임오버.
- **난이도 상승**: 웨이브당 몬스터 수 ↑, 이동 속도 ↑, 특수 몬스터(빠른/탱커) 등장.
- **장점**
  - 풀 수 있는 시간에 약간 여유가 있어 6~8세도 따라가기 좋음.
  - "히어로 = 성을 지키는" 정체성에 잘 부합.
- **단점**
  - A안과 비슷한 액션 게임 구조인데 **자산 요구가 더 많음**(여러 종류의 적 + 발사체 + 성).
  - 한 번에 보여줘야 할 정보가 많아 화면 구성이 빡빡.

---

### D. 두더지 잡기 (반응속도)

```
  🕳️   🕳️   🕳️
   👾
  [4×9]
─────────────
  🕳️   🕳️   🕳️

  [답: ___]
```

- **흐름**: 3×3 등 그리드에서 두더지가 문제와 함께 무작위로 튀어나옴 → 짧은 시간 안에 답 입력 → 망치로 처치.
- **실패 처리**: 시간 안에 못 풀고 두더지가 다시 들어가면 라이프 -1.
- **난이도 상승**: 두더지 등장 빈도 ↑, 등장 시간 ↓, 동시 등장 수 ↑.
- **장점**
  - 매우 단순하고 직관적, 6세도 즉시 이해.
  - 자산이 작음(두더지 + 구멍 + 망치 이펙트).
- **단점**
  - "히어로" 테마와 연결이 가장 약함.
  - 반응속도 위주라 **계산 학습 비중이 가장 낮음**(이미 답을 외운 단순 곱셈에 유리).

## 추천 방향

**A. 몬스터 처치**를 1차로 권장.
이유:
1. "연산 히어로"라는 프로젝트명과 가장 자연스럽게 연결됨.
2. 시간 압박이 게임 룰 자체에 녹아 있어 별도 UI가 단순.
3. 몬스터-처치 루프는 아동 게임에서 검증된 형식이며, 콤보·웨이브로 확장 가능성이 크다.

## 구현 시 신규/수정 파일 (A안 기준 예상)

```
lib/app/
  data/
    models/
      action_session.dart       (신규: HP, wave, kills, combo Rx)
      monster.dart              (신규: 위치/속도/문제)
      session_mode.dart         (수정: action 추가)
    services/
      problem_generator.dart    (수정: generateForDigits(type?, digitsA, digitsB) 추가
                                 — type == null 이면 매 문제 무작위 연산)
  modules/
    action_select/              (신규: 공통 진입 선택 — 연산 5지선다 + 자릿수 5지선다)
      action_select_view.dart
      action_select_controller.dart   (마지막 선택값 SharedPreferences 복원/저장 포함)
      action_select_binding.dart
    action_game/                (신규: 액션 플레이 화면 — concept별 분기)
      action_game_view.dart
      action_game_controller.dart
      action_game_binding.dart
      widgets/monster_widget.dart
  routes/
    app_routes.dart             (수정: actionSelect, actionGame 추가)
    app_pages.dart              (수정: GetPage 2개 등록)
  modules/home/
    home_view.dart              (수정: 특별 모드 줄에 ⚔️ 게임 타일 추가)
    home_controller.dart        (수정: openAction() 메서드 추가 — actionSelect로 이동)
```

기존 `RecordService` / `SfxService` / `ProblemGenerator` 는 그대로 재사용.
SFX는 기존 `correct`/`wrong` 외에 `attack` / `defeat` / `gameover` 정도 추가 권장(없으면 기존 효과음 재활용 가능).

## 결정 필요 항목

1. **컨셉 확정**: A / B / C / D 중 하나 (또는 조합).
2. ~~**연산 선택 방식**: 단일 연산만 / 혼합 가능 / 레벨별 다름?~~ → **확정**: 공통 진입 선택에서 4개 연산 + 랜덤(혼합) 중 단일 선택, 자릿수 조합 5종 중 단일 선택.
3. **저장 정책**: `GameRecord`에 액션 모드 결과를 어떻게 보여줄지(처치 수? 웨이브? 콤보 최고?). 또한 `(gameType, digitsA, digitsB)`를 함께 저장해 기록 화면에 라벨 표기할지 결정.
4. **그래픽 자산**: 자체 일러스트 / Lottie / 이모지 + 간단한 도형 중 어느 수준까지 갈지.
5. **랜덤 모드 자릿수**: 랜덤 연산일 때도 자릿수 조합을 그대로 강제할지, 아니면 매 문제 연산에 맞춰 자릿수도 약간 가변(예: 곱셈은 3×3 너무 큼 → 자동 축소)으로 둘지.
